# Лабораторная работа №3
## Очередь задач: итераторы и генераторы

Проект реализует пользовательскую коллекцию `TaskQueue` для платформы обработки задач.  
Основная цель — показать работу протокола итерации, ленивых генераторов и потоковой обработки задач в Python.

---

## Возможности

- пользовательская коллекция `TaskQueue`;
- поддержка протокола итерации (`__iter__`, `__next__`);
- повторный обход очереди;
- ленивые фильтры по статусу и приоритету;
- генератор обработки задач;
- совместимость со стандартными конструкциями Python: `for`, `list()`;
- логирование основных этапов работы.

## Описание модулей

### `src/models.py`
Содержит:
- `TaskStatus` — статусы задач;
- `Priority` — приоритеты задач;
- `Task` — модель задачи.

### `src/iterator.py`
Содержит класс `TaskQueueIterator`, реализующий протокол итератора.

### `src/queue.py`
Содержит класс `TaskQueue`, который:
- хранит задачи;
- поддерживает повторную итерацию;
- предоставляет ленивые фильтры;
- реализует генератор обработки задач.

### `main.py`
Точка входа для демонстрации работы очереди и логов.

### `tests/`
Набор тестов для проверки корректности работы моделей, итератора, очереди и точки входа.

## Что демонстрирует работа

В проекте показаны следующие механизмы Python:

- различие между `Iterable` и `Iterator`;
- использование `__iter__()` и `__next__()`;
- корректное завершение итерации через `StopIteration`;
- повторный обход коллекции через создание нового итератора;
- ленивые вычисления через `yield`.
